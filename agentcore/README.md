# AgentCore Runtime – CDSS agents

This folder is used to deploy CDSS agents to **Amazon Bedrock AgentCore Runtime**, per [docs/agentcore-implementation-plan.md](../docs/agentcore-implementation-plan.md) and [docs/agentcore-next-steps-implementation.md](../docs/agentcore-next-steps-implementation.md).

## Purpose

- **Runtime** hosts CDSS agents (severity assessment, routing, patient/resource/scheduling/engagement logic) that can use AgentCore **Gateway** tools (MCP/Lambda). There is no separate “Triage” agent in the CDSS architecture; severity assessment is part of the Supervisor flow and aligns with Telemedicine MCP (Specialist escalation).
- **Observability** (tracing, CloudWatch) links to Patient_ID / Doctor_ID for audit (CDSS.mdc).

## Prerequisites

- AWS CLI configured; region where AgentCore is available (e.g. `us-east-1`, or `ap-south-1` if supported).
- [AgentCore Get Started](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/agentcore-get-started-toolkit.html) and [AgentCore Python SDK](https://github.com/aws/amazon-bedrock-agentcore-sdk-python).

## Creating agents in AWS

**Agents are not created by Terraform.** You must create them using the **AgentCore starter toolkit** or the **AWS Console**. Full steps:

- **[docs/agentcore-create-agents-in-aws.md](../docs/agentcore-create-agents-in-aws.md)** – Option A (toolkit: `agentcore create` → `agentcore launch`) and Option B (Console: upload ZIP, create agent, create endpoint).

### Quick path (toolkit)

1. Install: `pip install bedrock-agentcore-starter-toolkit bedrock-agentcore`
2. From project root: `cd agentcore/agent` then `agentcore create` (answer prompts).
3. Optionally replace or merge generated code with the existing `main.py` (CDSS stub).
4. Test locally: `agentcore dev`; then `agentcore launch` to deploy.
5. Copy the **Runtime ARN** from the output.
6. In `infrastructure/terraform.tfvars`: set `use_agentcore = true` and `agent_runtime_arn = "<arn>"`, then `terraform apply`.

### Minimal agent in this repo

- `agentcore/agent/main.py` – minimal entrypoint (hospitals + CDSS severity-assessment stub) compatible with Bedrock AgentCore SDK `@app.entrypoint`.
- `agentcore/agent/requirements.txt` – `bedrock-agentcore`.
- You can use this as-is for **Console** deployment (zip and upload) or merge with code generated by `agentcore create`.

## Deploying an agent

1. **Prepare agent code** under `agentcore/agent/` (e.g. entrypoint and dependencies per SDK). This repo includes a minimal `main.py` and `requirements.txt`.
2. **Deploy** using one of:
   - **Toolkit:** `cd agentcore/agent` → `agentcore create` (if needed) → `agentcore launch`. See [docs/agentcore-create-agents-in-aws.md](../docs/agentcore-create-agents-in-aws.md).
   - **Console:** Build a deployment zip (with deps for Linux arm64), upload to S3, then Bedrock → AgentCore → Host Agent → create endpoint. See same doc.
3. **Note the Runtime ARN** returned by the deploy or shown in the Console.
4. **Set in Terraform** so the CDSS Lambda can invoke this runtime:
   - In `infrastructure/terraform.tfvars` (gitignored), set:
     - `use_agentcore = true`
     - `agent_runtime_arn = "<the-arn-from-step-3>"`
   - Run `terraform apply` so the router Lambda receives `USE_AGENTCORE` and `AGENT_RUNTIME_ARN`.

## Gateway tools

The **Gateway** exposes tools (e.g. `get_hospitals`) implemented by the Lambda in `infrastructure/gateway_tools_src/`. Setup is done via:

```bash
python scripts/setup_agentcore_gateway.py $(cd infrastructure && terraform output -raw gateway_get_hospitals_lambda_arn)
```

See [docs/agentcore-gateway-manual-steps.md](../docs/agentcore-gateway-manual-steps.md).
